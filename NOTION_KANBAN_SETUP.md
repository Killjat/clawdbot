# Notion Kanban 看板设置指南

## 🎯 Notion访问链接
**立即开始**：https://notion.so

## 📋 完整设置步骤

### 第1步：创建工作空间

1. **访问Notion**：https://notion.so
2. **注册/登录**账号
3. **创建新页面**：点击左侧 "+" 按钮
4. **命名页面**：`🚀 Moltbot项目管理`

### 第2步：创建数据库

#### 创建主数据库
```
1. 在页面中输入 "/database"
2. 选择 "Table - Full page"
3. 命名：📋 任务管理数据库
```

#### 设置数据库属性
```
属性名称          类型        选项/配置
─────────────────────────────────────────
📝 任务名称       Title       主标题
🏷️ 状态          Select      Backlog/Analysis/Development/Testing/Done
🔥 优先级        Select      🔴高/🟡中/🟢低
👤 负责人        Person      团队成员
📅 截止日期      Date        包含时间
⏱️ 预估工时      Number      小时
🎯 任务类型      Select      🚀功能/🐛缺陷/📚文档/🔧重构
📊 故事点        Select      1/2/3/5/8/13
🔗 依赖关系      Relation    关联其他任务
🚫 阻塞原因      Text        阻塞描述
💰 业务价值      Select      高/中/低
📈 进度          Number      0-100%
🏷️ 标签          Multi-select 前端/后端/测试/部署
📝 描述          Text        详细说明
```

### 第3步：创建Kanban视图

#### 添加看板视图
```
1. 点击数据库顶部的 "Add a view"
2. 选择 "Board"
3. 命名：🔄 开发看板
4. 按 "状态" 分组
```

#### 配置看板列
```
📋 Backlog (待办)
├── 颜色：灰色
├── WIP限制：无限制
└── 描述：待分析和规划的任务

🔍 Analysis (分析中)
├── 颜色：蓝色
├── WIP限制：2个
└── 描述：需求分析和技术设计

💻 Development (开发中)
├── 颜色：黄色
├── WIP限制：3个
└── 描述：正在编码实现

🧪 Testing (测试中)
├── 颜色：橙色
├── WIP限制：2个
└── 描述：功能测试和验证

✅ Done (完成)
├── 颜色：绿色
├── WIP限制：无限制
└── 描述：已完成并交付
```

### 第4步：设置过滤器和排序

#### 看板过滤器
```
基础过滤器：
- 状态 ≠ 已归档
- 负责人 = 当前用户（个人视图）

高级过滤器：
- 优先级 = 高（紧急任务视图）
- 任务类型 = 缺陷（Bug修复视图）
- 截止日期 ≤ 本周（本周任务视图）
```

#### 排序规则
```
1. 优先级：🔴高 → 🟡中 → 🟢低
2. 截止日期：最近的在前
3. 创建时间：最新的在前
```

## 🔧 自动化设置

### Notion自动化规则

#### 规则1：任务状态自动更新
```
触发条件：进度 = 100%
自动操作：状态 → ✅ Done
```

#### 规则2：逾期任务提醒
```
触发条件：截止日期 < 今天 且 状态 ≠ Done
自动操作：添加 🚨 逾期标签
```

#### 规则3：WIP限制提醒
```
触发条件：Development列 > 3个任务
自动操作：发送Slack通知
```

### 模板和公式

#### 自动计算剩余时间
```
公式：dateBetween(prop("截止日期"), now(), "days")
显示：距离截止还有X天
```

#### 任务完成率
```
公式：prop("进度") / 100
格式：百分比显示
```

#### 优先级排序分数
```
公式：
if(prop("优先级") == "🔴高", 3,
   if(prop("优先级") == "🟡中", 2, 1))
```

## 🔄 工作流程

### 每日工作流程

#### 晨会准备（9:00 AM）
```
1. 打开Notion看板
2. 检查昨日完成任务
3. 查看今日计划任务
4. 识别阻塞问题
5. 更新任务状态
```

#### 工作进行中
```
1. 移动任务到相应列
2. 更新进度百分比
3. 添加工作备注
4. 标记阻塞问题
5. 协调依赖任务
```

#### 日终总结（6:00 PM）
```
1. 更新任务完成状态
2. 记录明日计划
```
